### Переменные окружения

| Переменная | Обязательная | Назначение |
|---|---|---|
| `EXPO_PUBLIC_OPENAI_API_KEY` | Нет | Включает редактирование изображений через OpenAI `gpt-image-1`; при отсутствии используется Rork Toolkit |
| `EXPO_PUBLIC_TOOLKIT_URL` | Нет | Базовый URL Rork Toolkit (по умолчанию: `https://toolkit.rork.com`) |

Переменные также хранятся в EAS (окружение "production") и автоматически подтягиваются при сборке.

### Архитектура

#### Управление состоянием — два уровня

- **`providers/AppDataProvider.tsx`** — основные данные приложения: проекты, шаблоны, `lastActiveProjectId`. Использует `@nkzw/create-context-hook` для генерации компонента-провайдера и хука `useAppData()` из одной фабрики. Все мутации проходят через `updateProject(id, updaterFn)` или именованные хелперы. Автоматически сохраняет данные в AsyncStorage при изменении состояния (пропускает запись во время начальной гидратации через флаг `isHydrating`).
- **`stores/creditsStore.ts` / `stores/licenseStore.ts`** — Zustand-сторы для баланса кредитов и статуса лицензии. Оба загружаются в `app/_layout.tsx` до скрытия splash-экрана.

#### Навигация

Плоский Stack на `expo-router`. Динамические экраны проекта находятся в `app/project/[projectId]/`. Параметры маршрута всегда строки или массивы строк — используй `getSingleParam()` из `utils/routes.ts` для безопасного извлечения.

#### Пайплайн генерации изображений

`services/ai/imageGeneration.ts` запускает варианты параллельно через `Promise.all`. Для каждого варианта:
1. Вызывает `services/ai/promptBuilder.ts` для сборки промпта через `getVariantStrategies()` и `buildVariantPrompt()`.
2. Сначала пробует OpenAI `/v1/images/edits` (если задан `EXPO_PUBLIC_OPENAI_API_KEY`).
3. При ошибке или отсутствии ключа — фолбэк на Rork Toolkit `/images/edit/`.
4. Декодирует полученный base64 и сохраняет как локальный файл через `services/storage/fileStorage.ts` (`persistBase64Image`).

Параметр `mode` (`"free"` | `"pro"`) управляет качеством и количеством вариантов. Для `sketch` используется `input_fidelity: "low"`, для `photo` — `"high"`. Кредиты списываются только при `"pro"`-режиме без лицензии, _после_ успешной генерации.

Большие изображения перед отправкой ресайзятся: длинная сторона ≤ 1536px, JPEG качество 0.8, через `expo-image-manipulator`. Для фото ≤ 1536px по обеим сторонам ресайз не применяется.

#### Система кредитов

Полностью клиентская, хранится в AsyncStorage. Стоимость: low=1, medium=2, high=4 кредита за вариант. Логика в `services/storage/creditsService.ts`; Zustand-стор оборачивает её для React.

Магазин (`app/shop.tsx`) сейчас в режиме заглушки — кнопка покупки не активна, `addCredits` не вызывается. Оставить так до реализации настоящих покупок.

#### Система лицензий

Полностью офлайн, на основе контрольной суммы FNV-1a. Лицензионные ключи имеют формат `RAI-XXXX-XXXX-XXXX-XXXX` (16 hex-символов после удаления префикса и дефисов). Валидация и активация — в `services/license/licenseService.ts`. Генератор ключей на Python — `tools/keygen.py`. HTML-генератор — `keygen.html` на ПК пользователя. 

**КРИТИЧНО:** `LICENSE_SECRET` (`renderai-mf-2026-secret-key`) должен совпадать в `tools/keygen.py` и `services/license/licenseService.ts`. Изменение секрета сломает все ранее выпущенные ключи.

Активный ключ = безлимит генераций; без ключа = система кредитов.

#### Локальное хранилище изображений

Исходные фото и сгенерированные варианты сохраняются как локальные файлы через `expo-file-system`. При удалении проекта все связанные файлы очищаются через `deleteImageIfNeeded()` в `services/storage/fileStorage.ts`.

#### Соглашения по UI

- `constants/theme.ts` — единственный объект `appTheme` для цветов, радиусов и типографики. Импортировать как default export.
- `components/ui/Screen.tsx` — использовать `AppViewScreen` (с прокруткой) или `AppScreen` (фиксированный) как корневую обёртку для всех экранов.
- Все строки интерфейса — на русском языке.
- Чипы с выбором используют `toggleSnippet` (а не `appendSnippet`) для возможности снятия выбора повторным нажатием.
